# [Deliverable 2] 의사결정 라운드 저널 (Decision-Making Journal)

본 문서는 'MCP 기반 Intelligent CS Wiki' 프로젝트를 기획하고 설계하는 과정에서 인간 관리자(Human)와 AI 에이전트(Agent) 간에 이루어진 주요 의사결정 과정을 시간 순서대로 기록한 저널이다.

## Round 1: 지식 도메인 확정 (지식의 파편화 방지)
- **일시**: 2026-06-14
- **안건**: 위키 지식 베이스의 대상 도메인 선정
- **논의 내용**: 
  - 최초에는 강의 주제(CSE-3308)를 대상으로 검토하였으나, 특정 강의에 매몰될 경우 지식의 확장성이 제한되고 평가 기준이 협소해질 수 있다는 우려가 제기됨.
  - 지식 간의 유기적 연결을 보여주기에 가장 적합한 모델이 무엇인지 논의함.
- **결정 사항**: 
  - 학부 과정의 핵심인 **운영체제(OS), 컴퓨터 네트워크, 컴퓨터 보안** 3대 전공을 통합 도메인으로 선정.
  - 세 과목을 관통하는 '상호 참조 네트워크 지식 그래프'를 구축하여 위키의 본질적인 가치를 증명하기로 합의.

## Round 2: 멀티 에이전트 역할 분담 및 아키텍처 설계
- **일시**: 2026-06-14
- **안건**: 위키 파이프라인 엔진 및 검수 프레임워크 설계
- **논의 내용**: 
  - 단일 LLM 에이전트가 문서 생성과 검수를 동시에 수행할 경우, 자신의 오류를 인지하지 못하는 '자기 확증 편향' 및 '환각(Hallucination)' 현상이 발생할 위험을 인지함.
  - 상호 보완적인 멀티 에이전트 구조의 필요성을 검토함.
- **결정 사항**: 
  - **Gemini 에이전트**: 대규모 컨텍스트 분석, 엔티티 추출, 문서 간 자동 링크 생성을 담당하는 '인덱서' 역할 수행.
  - **Claude 에이전트**: 스키마 준수 여부(Linter) 및 기술적 논리 결함을 검수하는 '검증자' 역할 수행.
  - Subprocess API 연동(`main.py`)을 통한 하이브리드 자동화 구조 확정.

## Round 3: MCP(Model Context Protocol) 및 확장 도구 기획
- **일시**: 2026-06-14
- **안건**: 에이전트의 시스템 제어 권한 및 외부 데이터 연계 도구 정의
- **논의 내용**: 
  - 단순한 텍스트 처리에서 벗어나, 에이전트가 실제 파일 시스템을 안전하게 제어하고 외부의 실시간 데이터를 지식 베이스에 주입할 수 있는 인터랙티브한 구조가 필요함을 논의함.
- **결정 사항**: 
  - 로컬 위키 파일 제어를 위한 `read_wiki`, `write_wiki` MCP 툴 규격 정의.
  - 보안 도메인의 특수성을 살려 외부 취약점 데이터베이스를 조회하는 `cve_searcher` 툴을 PRD에 명시하고, 이를 통해 지식의 최신성을 유지하기로 결정.

## Round 4: 데이터 거버넌스 및 보안 정책 수립
- **일시**: 2026-06-14
- **안건**: AI 자동화에 따른 데이터 유실 방지책 및 편집 권한 설정
- **논의 내용**: 
  - 에이전트에게 강력한 쓰기 권한을 부여할 경우, 인간 관리자가 작성한 고유의 통찰이나 주석이 자동 업데이트 과정에서 덮어씌워질(Overwrite) 위험이 크다는 점을 확인.
- **결정 사항**: 
  - **Human Protected Section** 도입: 마크다운 내부에 `<!-- [Human Protected: Start] -->` 태그를 사용하여 에이전트의 쓰기 접근을 기술적으로 제한.
  - '인간 우선 원칙'에 기반한 데이터 거버넌스 규칙을 수립하여 에이전트의 자동화와 인간의 창의적 개입 간의 균형을 맞춤.

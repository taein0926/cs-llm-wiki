# [Deliverable 1] 지식 도메인 정의 문서 (Knowledge Domain Definition)

## 1. 타겟 지식 도메인 (Target Knowledge Domains)
본 프로젝트는 컴퓨터공학(CS)의 핵심 인프라를 구성하는 아래 **3대 필수 전공 과목**을 타겟 지식 도메인으로 선정한다.

- **컴퓨터 네트워크 (Computer Networks)**
- **운영체제 (Operating Systems)**
- **컴퓨터 보안 (Computer Security)**

## 2. 도메인 선정 배경 및 이유
본 도메인은 단순한 정보의 나열을 넘어, 지식 간의 **유기적 결합과 상호 참조(Cross-Reference)**가 가장 활발하게 발생하는 영역이다. 선정의 구체적인 이유는 다음과 같다.

### 2.1 개념적 유기성 및 상호 참조 네트워크
일반적인 RAG(Retrieval-Augmented Generation) 시스템이 단편적인 키워드 검색에 의존하는 것과 달리, 위키(Wiki) 시스템은 문서 간의 연결성을 통한 '지식 그래프' 구축이 핵심이다. 선정된 세 과목은 다음과 같은 강력한 연결 고리를 가진다.
- **OS의 프로세스/메모리 구조** ➔ **보안의 버퍼 오버플로우 취약점**: 운영체제의 메모리 관리 메커니즘을 이해해야만 보안상의 메모리 오염 공격을 분석할 수 있다.
- **OS의 소켓 관리** ➔ **네트워크의 전송 계층**: 운영체제의 시스템 콜과 커널 영역의 데이터 처리는 네트워크 통신의 성능과 안정성을 결정짓는다.
- **네트워크 보안(SSL/TLS)** ➔ **컴퓨터 보안**: 네트워크를 통해 전달되는 데이터의 무결성과 기밀성을 보장하기 위해 보안 프로토콜과 암호학적 원리가 결합된다.

### 2.2 위키 구축의 이상적 모델
위와 같은 연쇄적 관계는 위키 고유의 **'상호 참조 네트워크 그래프'**를 구축하기에 가장 이상적인 환경을 제공한다. 한 문서 내의 개념이 다른 과목의 문서로 뻗어나가는 구조를 통해 사용자는 파편화된 지식이 아닌 통합된 CS 통찰력을 얻을 수 있다.

## 3. 기대 효과
본 위키 시스템을 통해 다음과 같은 가치를 창출하고자 한다.

- **인간-LLM 협업 지식 관리**: 인간은 핵심 개념의 주석과 도메인 특화 지식을 제공하고, LLM 에이전트는 복잡한 관계를 매핑하고 최신 정보를 업데이트하며 구조를 유지하는 공생적 지식창고를 구축한다.
- **지식의 자가 증식 및 정교화**: 상호 참조 링크가 늘어날수록 지식의 밀도가 높아지며, 새로운 기술이나 보안 취약점이 발견되었을 때 기존 네트워크 내에서 그 영향력을 즉각적으로 파악할 수 있는 동적인 플랫폼 역할을 수행한다.
- **교육적 가치**: 학부생 및 엔지니어들에게 전공 과목 간의 벽을 허물고, 실무에서 마주하는 복합적인 기술적 문제를 다학제적(Interdisciplinary) 관점에서 바라볼 수 있는 가이드를 제공한다.

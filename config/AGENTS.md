# AGENTS: Human-LLM Collaboration Protocol

본 문서는 인간과 LLM 에이전트가 마크다운 기반 위키를 공동으로 유지보수할 때 준수해야 할 협업 원칙을 정의합니다.

## 1. 기본 원칙
- **상호 보완**: 인간은 위키의 방향성과 창의적 통찰을 제공하며, LLM은 대규모 데이터의 정제, 구조화 및 정합성 검사를 담당한다.
- **추적 가능성**: 모든 자동화된 변경 사항은 문서 내 'LLM Maintenance Log'에 기록되어야 한다.

## 2. 데이터 우선순위 및 충돌 해결
- **Human-First Policy**: 동일한 데이터 필드나 문단에 대해 인간이 직접 수정한 내용과 LLM이 자동 생성한 내용이 충돌할 경우, **인간이 작성한 레이어(Layer)에 절대적인 우선순위**를 둔다.
- **LLM의 제약**: LLM은 인간이 명시적으로 `[Human Protected]` 태그를 달아둔 구역을 절대 수정하거나 덮어쓰지 않는다.
- **Conflict Handling**: 충돌 발생 시 LLM은 해당 구역을 건드리지 않고, 별도의 로그나 주석을 통해 인간에게 검토를 요청한다.

## 3. 에이전트 역할 분담
- **Gemini Agent**: 대규모 원천 데이터(Raw Source)의 컨텍스트를 파악하고, 위키 항목 간의 관계를 인덱싱하는 '지식 채굴기' 역할을 수행한다.
- **Claude Agent**: 생성된 문서의 논리적 결함을 찾고, 마크다운 스키마 준수 여부를 확인하는 '품질 검사관' 역할을 수행한다.

## 4. 커뮤니케이션 규칙
- 에이전트는 문서 수정 시 항상 `Version` 번호를 갱신하며, 수정 이유를 명확히 기재한다.
- 인간 사용자는 `config` 폴더의 지침을 수정함으로써 에이전트의 행동 방식을 제어할 수 있다.
